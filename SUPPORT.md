# Support

## Ask a question

Use [GitHub Issues](https://github.com/Pythoughts-labs/claude-architect/issues)
for reproducible bugs and support questions. If GitHub Discussions is enabled
for the repository, use it for open-ended usage and design questions. Search
existing issues and discussions before opening a new one.

Include:

- the plugin version or commit;
- operating system and architecture;
- Claude Code, Node.js, and Git versions;
- selected Producer CLI and version;
- the command or workflow you ran and the expected versus actual result;
- minimal reproduction steps; and
- redacted diagnostics, logs, or error output.

Remove credentials, tokens, personal data, proprietary source, prompts, and
other secrets. Support is provided by maintainers and community members on a
best-effort basis; response times are not guaranteed.

## Security and conduct reports

Do not disclose suspected vulnerabilities in a public support thread. Follow
the private reporting process in [SECURITY.md](SECURITY.md). Report Code of
Conduct incidents only through the private channels listed in
[CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md), never through a public issue.
